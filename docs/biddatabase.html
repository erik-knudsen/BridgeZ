Bid Database.

The intention of the bid database is to identify the best next bid. This is based on selected conventions, the auction (bidding sequence) till now and the given hand (13 cards).

The database has three levels:

  •  The first level (page) selects a top level of rules, based on selected conventions. The first level is a one to one mapping (injective).

  • The second level (auction) delimits the rules in the top level set, to rules relevant for a given auction (bidding sequence). The second level is also a one to one mapping. 

  • The third level (next bid) further delimits the rules to one rule based on the hand (13 card) features (High card points etc.) and in some cases also to selected conventions. The selected rule identifies the next bid. The third level is a one to many mapping (the next bid can be selected by more than one rule). In some cases the third level instead identifies a substitute auction (sequence of bids). It might also happen that the database cannot find a best next bid. In which case the computer uses an algoritmic approach for finding the next bid.

In the following dialogs you can create/examine the bid database.

Bidding System.


The "Pages" box allows you to create or examine or edit or delete a page, the first level of the bid database. The Page Id identifies the page and is used by the bidding algorithms for including or excluding the page in the determination of the next bid. A textual description of the page is also shown for examination or for editing:



The "Auctions" box in the "Bidding System" dialog allows you to create or examine or edit or delete an auction, the second level of the bid database.
The "Link" box allows you to create or examine or edit or delete a link (a substitute auction). If an auction on the second level has a link to another auction then the link is substituted for the original auction.
The "Rules" box allows you to create or examine or edit or delete a next bid, the third level of the bid database.

The rules are described by the following dialogs.

Rule.



Here parameters for the rule is shown and can be edited.
The Rule Id identifies the rule and is used by the bidding algorithms for including or excluding the rule in the determination of the next bid. A Rule Id of 0 means that it is always included.
The Rule Priority is used when a number of rules are chosen by the bidding algorithm as equal valid. The rule with the highest priority is then chosen. In cases where the Rule Id is not 0 you can write a textual description of the rule in the Rule Id Description box. 
The Alert Id identifies the alert and must be not 0 for rules which must be alerted. In cases where the Alert Id is not 0 you can write a textual description of the alert in the Alert Id Description box.

High Cards and Quality.



Range for High Cards and Quality can be defined in this dialog. Range is defined for each suit. S for Spades, H for Hearts, D for Diamonds, C for Clubs and A for all. Range is defined as [from]-[to], [from]+ or +[to]. Where [from] and [to] are numbers. The number is included in the range.

The following parameters are defined in the dialog (max range is shown with suits first and all second):

  • HCP, weight of High Cards. Ace counts 4, King counts 3, Queen counts 2, Jack counts 1. Range [0-10], [0-37].
  • Controls, weight of High Cards. Ace counts 2, King counts 1. Range [0-3], [0-12].
  • HC, count of aces, kings and queens.  Range [0-3], [0-12]
  • Honor, count of cards higher than 9. Range [0-5], [0-13]
  • Ace, count of aces. Range [0-1], [0-4]
  • King, count of kings. Range [0-1], [0-4]
  • Queen, count of queens. Range [0-1], [0-4]
  • Jack, count of Jacks. Range [0-1], [0-4]
  • Ten, count of Tens. Range [0-1], [0-4]
  • RKCB, count of aces + king in the suit. Range [0-5].
  • Inter, weighed count of intermediate cards. 10 counts 4, 9 counts 2, 8 counts 1. Range [0-7], [0-28]
  • Looser, loosers. Only the first three cards in the suit are potential losers. Ace King and Queen are not loosers. Range [0-3], [0-12]
  • Quick, quick tricks. Only Ace, King and Queen counts. Range [0-4], [0-16]. Multiplied with 2.
  • Play, playing tricks.  Range [0-26]. Multiplied with 2.
  • StopNT, stoppers in NT for a given suit. Range [0-4].
  • Qlty, general quality of cards for a given suit. Range [0-6].
  • Points, HCP + distribution points + lenght points. Range [0-37], [0-37].
  • Dist, distribution points. Double counts 1, single counts 2, renonce counts 3. Range [0-9][0-9].

Length.


This dialog shows length ranges, which are straightforward. The "Longest" column in the "Length" box is 1 if the suit is the longest, 0 otherwise. The "Long" column in the "Length" box is 1 if the suit is not the shortest. The parameters in the Other box:

  • Balanced, 0 = balanced, 9 = freak.
  • Longer Than 3, number of suits longer than 3.
  • Longer Major, length of longest major.
  • Longer Minor, lenght of longest minor.
